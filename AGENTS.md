# AGENTS guidance for common-instructions

Scope: entire repository.

## Purpose
This repo contains shared instruction artifacts used across org repositories.

## Rules
1. Keep all deliverables at repository root-level directories defined in README.
2. Do not create nested org/repo scaffolding inside this repository.
3. Do not store copied vision/org source documents here; reference external paths instead.
4. Keep prompts generic and reusable across platform orgs.
5. Update `_index.yaml` to match actual files whenever structure changes.
6. Add/update a review note under `reviews/` for substantial execution runs.

## Content quality bar
- Use explicit boundaries (`in-scope` and `out-of-scope`) in prompt files.
- Keep templates concise and execution-oriented.
- Avoid platform-specific assumptions unless a file is explicitly marked for that platform.
