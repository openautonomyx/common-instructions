# Terms Summary

## Open-Core Licensing Model
- Core application code is open source.
- `common-instructions`, premium guardrails, and continuous improvement loops are closed source.
- Customers may self-host using their own API keys.

## Commercial Boundary
- Payment is for premium instruction assets, managed updates, and new feature delivery.
- Access to premium assets is controlled by active subscription tier.
- Air-gapped operation capability is a premium entitlement.

## Usage Rules
- Community users can run open-source components without premium update rights.
- Free tier has capped tool/skill access and does not include policy enforcement or SSO.
- Paid tiers can access private registries and premium update channels per plan.
- Air-gapped/offline enterprise bundle is available only on Enterprise tier unless separately contracted.
- Redistribution of closed-source instruction assets is prohibited without explicit commercial agreement.

## Support Scope
- Community: best-effort/public support.
- Pro/Team/Enterprise: tier-defined support and response expectations.

## Compliance and Security
- Secrets must be managed in a secret manager.
- CI/CD-only deploy policy for production.
- Air-gapped support is available in Enterprise or custom contract.

## Legal Notes
- This document is a concise operational summary and not full legal terms.
- Final contractual terms are defined by MSA/order form/license agreement.
