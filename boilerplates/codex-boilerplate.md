# Codex Execution Boilerplate

## Objective
State the task objective in one sentence.

## Inputs
- Canonical prompt path
- Required reference docs
- Repo constraints

## Workflow
1. Verify repo reality.
2. Resolve conflicts via source-of-truth order.
3. Make smallest viable changes.
4. Validate outputs.
5. Document results and deferred items.

## Guardrails
- Do not invent missing source-of-truth inputs.
- Prefer explicit file-level updates.
- Keep boundaries clear (shared vs platform-specific).

## Output Contract
- Summary
- Verified current state
- Files created/updated
- Boundary decisions
- Risks/conflicts
- Deferred items
- Common-layer improvements
